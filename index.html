HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • 'Child' elements nest within 'Parent' elements.
  • Always use headers in order, and only use h1 once per page, for the sake of accessibility.

CSS

  • Margin: indicates how much space we want around an the outside of an element
  • Padding: how much space we want aroudn the content inside an element. Adding more padding can make an element bigger
  • Using an external CSS style sheet means you only have to change one file to change multiple pages that share the same style sheet.
  • You can add multiple shadows to the same element by seperating the details with a comma.

Git

  • git status: checks our current branch.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git checkout branch-name: moves to branch 'branch-name'.
  • git add -a: commit all files in the working branch. replace -a with single file, if needed.
  • git commit -m "commit message": commits branch code, and message should be short and descriptive.

JavaScript

  • Javascript files need to be linked to the HTML document to be used. Always add them at the end of the document because HTML gets processed in order, and JS provides the logic for HTML. If the logic is processed before the content, errors may occur.
  • Javascript can output to the console via console.log(variable name), which is vital for debugging.
  • Javascript uses loops and arrays and all sorts of fancy data types
  • Control flow is the order in which a computer executes code in a script
  • Variables are buckets, but unlike buckets they need useful names.
